windows服务器安全加固记录

公司有windows server 2012服务器一台，常年处于裸奔状态。

服务器现状

  1. 服务器连接公网（不知道啥原因不能断）
  2. 防火墙关闭状态
  3. 未打过任何补丁
  4. 跑着业务软件

对于攻击者来说是一只很香的肉鸡了。

解决办法

  1. 检查业务软件的端口

    运行CMD - 在窗口中输入”netstat -a -n”

    找到业务端口后添加到防火墙规则中

  2. 开启防火墙清理无用规则

    关闭不必要的端口

    添加关闭135、137、138、139、445端口规则，防止永恒之蓝漏洞

  3. 开启自动更新，定期安装最新补丁

  4. 密码定期修改，网上有个密码生成器，可以生成超复杂的随机密码，估计爆破不出来的。
